# 回复归档与存储调研

## 1. 设计目标

- **人可读**：Markdown 格式，可在 GitHub 直接浏览
- **机器可查**：结构化 front-matter，便于脚本过滤和统计
- **增量追加**：新回复只新增文件/追加内容，不破坏已有记录
- **版本追踪**：Git commit 历史即审计日志

## 2. 目录结构设计

```
archive/
├── articles/
│   └── {article_id}/
│       ├── meta.md              # 文章标题、URL、监控开始时间
│       └── comments/
│           └── {comment_id}.md  # 每条评论及机器人回复
data/
├── seen_ids.json                # 已处理评论 ID 集合
├── wiki_hash.json               # CSM Wiki 文件哈希
└── vector_store/                # ChromaDB 持久化目录
```

## 3. 单条评论存档格式

文件：`archive/articles/{article_id}/comments/{comment_id}.md`

```markdown
---
comment_id: "12345678"
article_id: "98765432"
article_title: "CSM 最佳实践系列（一）"
article_url: "https://zhuanlan.zhihu.com/p/98765432"
commenter: "用户昵称"
comment_time: "2024-04-09T10:30:00+08:00"
replied_at: "2024-04-09T10:35:12Z"
model: "gpt-4o-mini"
tokens_used: 342
wiki_refs:
  - "csm-wiki/onboarding.md#第一节"
  - "csm-wiki/best-practices.md#客户沟通"
---

## 原评论

> 请问 CSM 如何处理客户的升级投诉？

## 机器人回复

针对客户升级投诉，建议遵循以下步骤：首先在 24 小时内确认收到...（略）

## 参考来源

- csm-wiki/onboarding.md § 第一节
- csm-wiki/best-practices.md § 客户沟通
```

## 4. 文章元数据文件

文件：`archive/articles/{article_id}/meta.md`

```markdown
---
article_id: "98765432"
title: "CSM 最佳实践系列（一）"
url: "https://zhuanlan.zhihu.com/p/98765432"
monitored_since: "2024-04-01"
total_replies: 12
last_checked: "2024-04-09T10:35:00Z"
---
```

## 5. Git 提交策略

```bash
# 每次 Action 运行结束后提交
git add archive/ data/seen_ids.json
git diff --cached --quiet || \
  git commit -m "bot: reply to $(git diff --cached --name-only | wc -l) new comments [skip ci]"
git push
```

- `[skip ci]` 避免存档提交再次触发 workflow
- commit message 包含本次处理数量，便于快速查阅历史
- 向量库（`data/vector_store/`）可单独 commit 或用 `.gitignore` 排除并缓存于 Actions Cache

## 6. 状态文件：seen_ids.json

```json
{
  "articles": {
    "98765432": ["12345678", "12345679"],
    "98765433": ["22222222"]
  },
  "last_run": "2024-04-09T10:35:00Z"
}
```

- 轻量，只存 ID 字符串
- 每次 Action 读取 → 比对 → 更新 → commit

## 7. 检索与统计

由于每条回复都是独立 Markdown 文件，可用 shell/Python 快速统计：

```bash
# 统计总回复数
find archive/ -name "*.md" ! -name "meta.md" | wc -l

# 查询某文章的所有回复
ls archive/articles/98765432/comments/

# grep 关键词
grep -r "升级投诉" archive/
```
