# 知乎数据获取调研

## 1. 官方 API 现状

知乎**没有公开的官方 API**，但其网页端使用 RESTful v4 接口，可通过以下两种方式访问：

| 方式 | 认证 | 适用场景 | 风险 |
|------|------|----------|------|
| Cookie（z_c0 + \_xsrf） | 浏览器登录后抓包导出 | 个人/小规模 | 账号封禁风险 |
| OAuth 2.0 | 开放平台注册应用 | 正式/高频 | 需审核，有限配额 |

> 推荐路径：使用 Cookie 认证做 PoC，后期如有必要再申请 OAuth。

## 2. 核心读取接口（API v4）

```
# 专栏文章评论（分页，最大 limit=20）
GET https://www.zhihu.com/api/v4/articles/{article_id}/comments
    ?limit=20&offset=0

# 问题下回答列表
GET https://www.zhihu.com/api/v4/questions/{question_id}/answers

# 回答下评论
GET https://www.zhihu.com/api/v4/answers/{answer_id}/comments

# 用户信息
GET https://www.zhihu.com/api/v4/members/{url_token}
```

返回 JSON 包含 `data[]` 和 `paging.is_end`，需循环翻页直到 `is_end=true`。

参考：[知乎 API v4 整理](https://yifei.me/note/460) | [博客园整理版](https://www.cnblogs.com/apachecn/p/17313135.html)

## 3. 写操作接口

```
# 发表评论（一级 / 楼中楼回复加 parent_id）
POST https://api.zhihu.com/v4/comments
Body: { content, object_id, object_type: "article"|"answer", [parent_id] }
Header: Authorization: Bearer {access_token}

# 发布回答
POST https://api.zhihu.com/v4/answers
Body: { question_id, content }
```

> 写操作**必须**使用 OAuth access_token，Cookie 方式无法调用写接口。

## 5. 反爬与长期稳定访问

知乎反爬机制严格，核心策略：

| 问题 | 对策 |
|------|------|
| Cookie 过期 | 加密存储 Cookie 于 GitHub Secrets；设置检测逻辑，失效时告警（自动续签难度大） |
| 频率限制（429） | 请求间随机延迟 1~3s；指数退避重试；本项目为低频轮询，风险低 |
| IP 封禁 | GitHub Actions IP 每次不同，天然规避；如被封可切换 runner |
| 行为识别 | 带 User-Agent（浏览器 UA）、Referer；不并发抢，顺序读取 |

**关键：** Cookie 中 `z_c0` 是身份凭证，需作为 GitHub Secret 存储，**绝不提交到代码**。

## 6. 数据获取流程

```python
# 伪代码：增量评论获取
seen_ids = load_seen_ids("data/seen_comments.json")
for article_id in config["articles"]:
    comments = fetch_all_comments(article_id, cookie)
    new = [c for c in comments if c["id"] not in seen_ids]
    if new:
        process_and_reply(new)
        seen_ids.update(c["id"] for c in new)
save_seen_ids(seen_ids)
```

## 7. 速率限制参考

| 认证类型 | 大致限制 |
|----------|----------|
| Cookie（普通用户） | ~60–100 次/分钟 |
| OAuth 应用 | ~500–1000 次/分钟 |

本项目每次 Action 仅需读取少量文章评论，远低于限制，无需代理池。
